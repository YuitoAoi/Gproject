# 阶段 2 计划：数据管理模块

本计划概述了实现数据管理模块的可执行步骤，该模块是路线图的第二阶段。计划基于 `2-CONTEXT.md` 中定义的架构决策。

## 第 1 部分：后端实现

### 任务 1.1：文件上传 API 端点
- **动作**: 在 `backend/app/api/` 目录下创建一个新的路由文件，用于处理数据集相关的请求。
- **动作**: 实现 `POST /api/v1/datasets/initiate-upload` 端点。此端点应生成一个唯一的 `uploadId` 并将其与元数据（如文件名、总大小）存储在 Redis 中，设置一个合理的过期时间。
- **动作**: 实现 `POST /api/v1/datasets/upload-chunk` 端点。此端点应接收文件分块，并将其以 `uploadId_chunkNumber` 的格式临时存储在磁盘的特定上传目录中。
- **动作**: 实现 `POST /api/v1/datasets/complete-upload` 端点。此端点将触发一个 Celery 异步任务，以合并分块并完成文件处理。
- **验证**: 使用 API 测试工具（如 Postman 或 `curl`）可以成功调用这三个端点，并且临时分块文件和 Redis 记录都已正确创建。

### 任务 1.2：文件合并与数据持久化的异步任务
- **动作**: 创建一个新的 Celery 任务 `assemble_and_save_dataset`。
- **动作**: 在此任务中，实现逻辑以：
  1.  根据 `uploadId` 找到所有相关的分块文件。
  2.  按正确的顺序将它们合并成一个最终文件。
  3.  将合并后的文件移动到永久的数据集存储位置。
  4.  在 `datasets` 数据库表中创建一条记录，包含文件名、路径、大小等元数据。
  5.  清理临时分块文件和 Redis 中的 `uploadId` 记录。
- **验证**: 成功触发 `complete-upload` 端点后，Celery worker 会执行此任务。最终文件被正确创建，数据库记录也已插入，并且临时文件被删除。

### 任务 1.3：数据处理的异步任务
- **动作**: 创建 Celery 任务，用于数据清洗和格式转换（例如，`process_dataset_task`）。
- **动作**: 实现 `POST /api/v1/datasets/{dataset_id}/process` 端点。此端点将根据请求体中的 `processType` 来分派相应的 Celery 任务，并立即返回 `taskId`。
- **验证**: 调用此端点会成功触发一个 Celery 任务，并且 API 会立即返回一个 `taskId`，而不会等待任务完成。

### 任务 1.4：数据集 CRUD API
- **动作**: 实现 `GET /api/v1/datasets` 端点，用于从数据库中检索数据集列表。
- **动作**: 实现 `DELETE /api/v1/datasets/{dataset_id}` 端点，用于删除数据库记录和关联的文件。
- **验证**: 这些端点按预期工作，能够正确地列出和删除数据集。

## 第 2 部分：前端实现

### 任务 2.1：数据管理页面和路由
- **动作**: 在 `frontend/src/features/` 下创建一个 `data-management` 目录。
- **动作**: 在 `frontend/src/router/index.js` 中，为“数据管理”页面添加一条新路由 (`/data-management`)，并将其链接到侧边栏菜单。
- **动作**: 创建 `DataManagementView.vue` 作为该页面的主视图。
- **验证**: 在 UI 中点击“数据管理”菜单项，可以成功导航到空白的 `DataManagementView` 页面。

### 任务 2.2：数据集列表组件
- **动作**: 创建一个 `DatasetList.vue` 组件，使用 Element Plus 的 `el-table` 来展示数据集。
- **动作**: 在 `DataManagementView` 加载时，调用后端的 `GET /api/v1/datasets` 端点，并将获取到的数据填充到表格中。
- **动作**: 在表格的操作列中添加“删除”按钮，并实现调用 `DELETE` 端点的逻辑（包括确认对话框）。
- **验证**: 数据集列表可以正确显示。删除功能可以正常工作。

### 任务 2.3：文件上传组件
- **动作**: 创建一个 `DatasetUploader.vue` 组件。
- **动作**: 使用 Element Plus 的 `el-upload` 组件，并自定义其 `http-request` 以实现分块上传逻辑：
  1.  在上传开始前，调用 `initiate-upload` API。
  2.  将文件分块，并为每个分块调用 `upload-chunk` API。
  3.  监控所有分块的上传进度。
  4.  在所有分块上传完成后，调用 `complete-upload` API。
- **动作**: 在上传成功后，触发一个事件以刷新数据集列表。
- **验证**: 可以通过此组件成功上传一个大文件。上传进度条能正确反映实际进度。上传成功后，新文件会出现在列表中。

### 任务 2.4：数据处理功能
- **动作**: 在数据集列表的操作列中，为每个状态为 `Ready` 的数据集添加“处理”按钮（例如，“转换为 Alpaca 格式”）。
- **动作**: 点击按钮后，调用 `POST /api/v1/datasets/{dataset_id}/process` 端点。
- **动作**: 在 API 调用后，将表格中该行的状态更新为“处理中”，并禁用其操作按钮。
- **验证**: 点击按钮后，后端会收到请求并启动一个 Celery 任务。前端 UI 会立即更新为“处理中”状态。

## 第 3 部分：最终验证 (UAT)

### 任务 3.1：端到端数据生命周期测试
- **动作**: 执行一个完整的用户流程：
  1.  上传一个新的数据集文件。
  2.  在列表中查看该数据集。
  3.  触发一个数据转换任务。
  4.  （在后续阶段实现任务监控之前）在数据库中验证任务状态是否已更新。
  5.  删除该数据集。
- **验证**: 整个流程可以无差错地完成。所有后端和前端的状态都按预期进行了更新。
