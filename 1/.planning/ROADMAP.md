# 项目路线图 (ROADMAP.md)

## 里程碑 1: 项目初始化与基础架构搭建 (v1.0)
### 阶段 1: 前后端基础架构搭建
- **前端**:
    - 初始化 Vue 3 + Vite 项目，并建立“按功能/模块组织”的目录结构。
    - 实现“顶部导航 + 侧边菜单 + 内容区”的整体应用布局。
    - 集成并配置 Vue Router, Pinia。
    - 创建可复用的 API 通信模块 (HTTP Client & WebSocket Manager)。
- **后端**:
    - 配置 Python 环境并搭建 FastAPI, Celery, SQLAlchemy 基础服务。
    - 根据数据库设计，创建数据库表结构。
    - 集成 LLaMA-Factory 作为核心 LLM 框架。
- **UAT**:
    - 后端服务能够正常启动，并成功连接数据库。
    - 前端应用能够展示完整的页面布局，基础路由可正常跳转。

### 阶段 2: 数据管理模块
- 实现文件上传功能，包括大小限制、类型校验、进度显示。
- 开发数据清洗工具，支持预览和生成新数据集。
- 实现格式转换到 LLaMA-Factory 格式，生成 dataset_info.json。
- 构建数据集管理界面和操作 (列表、查询、删除、版本化)。
- UAT: 测试上传、清洗、转换流程，确保格式兼容 LLaMA-Factory。

## 里程碑 2: 训练与监控功能 (v1.1)
### 阶段 3: 模型训练任务编排
- 开发任务配置 Web 界面，支持参数输入校验和默认值。
- 实现资源预估逻辑，包含警告和强制启动选项。
- 设置任务队列和调度，使用 Celery 管理多任务排队。
- 添加任务状态更新和中止功能 (PID 管理、SIGTERM、资源清理)。
- **模型保存与注册**: 训练成功后，将最终模型权重保存到指定路径，并将其元信息注册到数据库中。
- **检查点支持 (可选)**: 支持按 Epoch 保存训练检查点以实现断点续训。
- UAT: 模拟训练任务提交、排队、中止，确保无资源泄漏。验证成功完成的任务会产出可用的模型文件和数据库记录。

### 阶段 4: 实时监控
- 集成日志流捕获和 WebSocket 推送。
- 解析并展示训练指标 (Loss 等) 以图表形式。
- 实现硬件监控 (GPU 指标) 使用 pynvml。
- 添加进度条和 Epoch/Step 显示。
- UAT: 验证实时性，低延迟传输，无数据丢失。

## 里程碑 3: 推理与可观测性 (v1.2)
### 阶段 5: 模型推理与服务
- 实现后台模型加载，显示进度和模型信息。
- 开发交互式聊天界面，支持多轮对话和流式输出。
- UAT: 测试模型加载和推理响应时间，确保流式输出顺畅。

### 阶段 6: 全链路可观测性
- 扩展硬件状态监控到持续展示。
- 集成任务生命周期视图。
- 确保日志和指标的详尽记录和实时传输。
- UAT: 验证全链路监控覆盖所有模块，无盲点。

## 里程碑 4: 测试、优化与部署 (v1.3)
### 阶段 7: 全面测试与优化
- 进行单元测试、集成测试和端到端测试。
- 处理边界情况、安全性检查和性能优化。
- 优化轻量级部署，支持本地/私有云。
- UAT: 模拟生产环境测试，确保稳定性。

### 阶段 8: 文档与发布
- 编写用户手册和 API 文档。
- 准备部署脚本和配置指南。
- 发布 v1.0 版本。
- UAT: 验证文档完整性，用户易用性。
