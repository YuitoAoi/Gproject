数据集处理使用方法
整体流程
Step1 选择源数据 → Step2 LLM配置 → Step3 执行任务
---
### Step1：选择源数据
1. 页面加载时自动从后端获取数据集列表（`getDatasets()` API）
2. 从列表中选择一个数据集
3. 点击 **「下一步：配置清洗规则」** 进入 Step2
---
Step2：LLM 配置（3 个折叠面板）
|面板|配置项|说明|
|----|----|----|
|1. LLM 配置|API Key / 服务地址 / 模型名称|调用 LLM 的凭证|
|2. 生成模式|Mode / 输出格式|atomic/multi_hop/CoT 等 8 种模式；Alpaca/Sharegpt/ChatML 三种格式|
|3. 高级设置|分词器 / 块大小 / 块重叠 / 每单元问答数 / 分区算法 / RPM / TPM	|性能调优参数  

右侧面板显示配置说明（后端 GraphGen 不支持实时预览）。
点击 「提交并开始异步清洗任务」 进入 Step3。
---
Step3：执行监控
左侧 - 任务大盘
- 任务状态：⏳等待中 → 处理中 → ✅已完成 / ❌失败
- 进度条（处理中时显示）
- 数据质量漏斗：原始数据 → 过滤清洗 → 相似度去重 → 最终产出
右侧 - 终端日志
- 模拟 GraphGen 流水线日志（LLM连接 → 图构建 → 问答生成 → 格式化输出）
- 状态每 3 秒轮询一次 getDatasetDetail() API，根据 dataset.status 推断任务进度
- status: 0=初始化, 1=处理中, 2=已完成, 3=失败
点击 「返回资产池」 结束监控，返回数据集管理页。
---
注意事项
- 关闭/跳转页面不会中断任务（任务在服务器端运行）
- api_key 安全问题暂未处理（直接输入明文）
- 当前 Step3 日志为 mock 模拟数据，后端 GraphGen 实际输出后需对接真实日志流
